## Что делаем

Проверяем не только запуск приложения, но и отдельные расчётные функции.

## Проверка стоимости дозакупки

Создайте `check_costs.py`:

```python
from decimal import Decimal

from calculations import calculate_min_purchase_cost

cases = [
    (1570, 5500, 30, Decimal("15.29")),
    (3000, 3000, 30, Decimal("17.6666667")),
    (500, 1500, 15, Decimal("76.59")),
]

for case in cases:
    print(case, "->", calculate_min_purchase_cost(*case))
```

Запуск:

```bat
python check_costs.py
```

Сравните результат с контрольными значениями своего варианта.

## Проверка метода модуля 4

Создайте `check_module4.py`:

```python
from calculations import calculate_products_count

checks = [
    (1, 1, 1000, 2.5, 1.5),
    (2, 4, 5000, 1.2, 2.1),
    (4, 5, 12000, 3.0, 0.8),
    (99, 1, 100, 1.0, 1.0),
    (1, 1, 100, -1.0, 2.0),
]

for check in checks:
    print(check, "->", calculate_products_count(*check))
```

Запуск:

```bat
python check_module4.py
```

## Ручная проверка UI

Проверьте сценарии:

- список материалов загружается;
- стоимость партии считается для каждой карточки;
- кнопка «Добавить материал» создаёт новую запись;
- редактирование меняет запись в БД;
- окно поставщиков открывается для выбранного материала;
- приложение не падает при некорректном вводе.

## Критерий готовности этапа

Расчёты совпадают с контрольными значениями индивидуального варианта, а основные сценарии интерфейса работают.
