Эта документация построена как конвейер. Нельзя пропускать этапы: каждый следующий шаг опирается на результат предыдущего.

## Что студент получит в конце

- рабочую базу данных MySQL;
- импортированные индивидуальные данные;
- Python-приложение на PyQt6;
- расчёт стоимости минимальной партии;
- расчёт количества продукции по модулю 4;
- формы просмотра, добавления и редактирования материалов;
- окно поставщиков выбранного материала;
- SQL-экспорт базы;
- ER-диаграмму;
- собранный `.exe`;
- локальный git-репозиторий с итоговым коммитом.

## Логика прохождения

1. Сначала готовится папка проекта и индивидуальные приложения с данными.
2. Затем поднимается MySQL и создаётся схема.
3. После этого данные импортируются и проверяются SQL-запросами.
4. Далее создаётся Python-приложение.
5. В конце выполняются проверки, сборка и подготовка итогового комплекта.

::: tip
Если у группы разные приложения с данными, документация остаётся общей. Меняются только файлы в папке `resources` и контрольные значения, которые выдаёт преподаватель.
:::
