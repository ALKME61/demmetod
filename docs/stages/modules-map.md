## Модуль 1. База данных

**Цель:** создать базу данных для подсистемы материалов, подготовить импорт, загрузить данные, получить SQL-скрипт и ER-диаграмму.

| Этап | Что закрывает |
|---|---|
| [2. Индивидуальные данные](./02-data-appendices.md) | Проверка выданных import-файлов и подготовка CSV |
| [3. MySQL и phpMyAdmin](./03-mysql-docker.md) | Среда для администрирования БД |
| [4. Схема базы данных](./04-database-schema.md) | Таблицы, связи, ключи, 3НФ |
| [5. Импорт данных](./05-import-data.md) | Загрузка CSV в таблицы |
| [6. Контроль базы данных](./06-database-checks.md) | Проверка строк, связей и готовности данных |
| [12. Git, SQL-экспорт и ER-диаграмма](./12-git-export.md) | SQL-скрипт и ER-PDF |

::: tip Результат модуля 1
После этих этапов есть рабочая БД, импортированные данные, SQL-экспорт и ER-диаграмма.
:::

## Модуль 2. Программный модуль

**Цель:** разработать приложение учёта материалов, подключить его к БД и реализовать расчёт стоимости минимальной партии.

| Этап | Что закрывает |
|---|---|
| [7. Python-окружение](./07-python-env.md) | Подготовка зависимостей |
| [8. Подключение к базе данных](./08-db-layer.md) | Связь приложения с MySQL |
| [9. Расчёты](./09-calculations.md) | Функция стоимости минимальной партии |
| [10. Интерфейс PyQt6](./10-pyqt-ui.md) | Главная форма, список материалов, стиль |
| [11. Проверка программного продукта](./11-product-checks.md) | Проверка запуска и расчёта |

::: tip Результат модуля 2
После этих этапов приложение показывает материалы из БД и рассчитывает стоимость минимально необходимой партии.
:::

## Модуль 3. Интерфейс, сопровождение и обработка ошибок

**Цель:** реализовать последовательный интерфейс, переходы между окнами, добавление/редактирование материала и понятные сообщения об ошибках.

| Этап | Что закрывает |
|---|---|
| [10. Интерфейс PyQt6](./10-pyqt-ui.md) | Форма добавления/редактирования, переходы, сообщения |
| [11. Проверка программного продукта](./11-product-checks.md) | Проверка сценариев пользователя и обновления списка |

::: tip Результат модуля 3
После этих этапов пользователь может добавлять и редактировать материалы без ручной работы с БД.
:::

## Модуль 4. Интеграция программных модулей

**Цель:** добавить поставщиков материала и метод расчёта количества продукции с учётом коэффициентов и потерь.

| Этап | Что закрывает |
|---|---|
| [9. Расчёты](./09-calculations.md) | Метод `calculate_products_count` |
| [10. Интерфейс PyQt6](./10-pyqt-ui.md) | Окно поставщиков выбранного материала |
| [11. Проверка программного продукта](./11-product-checks.md) | Проверка метода и окна поставщиков |
| [12. Git, SQL-экспорт и ER-диаграмма](./12-git-export.md) | Коммит метода модуля 4 |

::: tip Результат модуля 4
После этих этапов приложение показывает поставщиков конкретного материала, а код содержит отдельный метод расчёта количества продукции.
:::
